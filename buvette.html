Concept

Le projet vise à moderniser la gestion d’une Buvette en supprimant les paiements en espèces. Chaque client possède un compte rechargeable et peut payer directement via son solde. Les barmans disposent d’un tableau de bord pour gérer les commandes et les stocks.

Fonctionnalités principales

  • Création de comptes clients rechargeables
  • Gestion des commandes en temps réel
  • Suivi des stocks
  • Gestion des rôles utilisateurs (client, barman, admin)
  • Interface administrateur complète

Architecture technique

  • PHP natif côté serveur
  • Base de données MySQL
  • Requêtes SQL dynamiques
  • JavaScript pour les interactions
  • Bootstrap pour l'interface responsive
